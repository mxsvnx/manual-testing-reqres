# Тест-кейсы — форма авторизации Reqres.in

**Объект:** POST https://reqres.in/api/login  
**Тип тестирования:** Функциональное, ручное  
**Версия:** 1.0

---

## TC-001 — Успешная авторизация

| Поле | Значение |
|------|----------|
| **ID** | TC-001 |
| **Название** | Успешный вход с валидными данными |
| **Приоритет** | Высокий |
| **Тип** | Позитивный |
| **Предусловие** | Пользователь `eve.holt@reqres.in` зарегистрирован в системе |

**Шаги:**
1. Отправить POST-запрос на `https://reqres.in/api/login`
2. Тело запроса: `{"email": "eve.holt@reqres.in", "password": "cityslicka"}`
3. Заголовок: `Content-Type: application/json`

**Ожидаемый результат:**
- Статус-код: `200 OK`
- Тело ответа содержит поле `token` с непустым строковым значением
